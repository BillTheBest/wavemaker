Data objects on palette become instances of data components, essentially a LiveVariable preset to a type.

Data components behave as now in inspector: sub-objects are shown in the inspector-tree, fields are shown in the inspector proper for the seleted data-object or sub-object.

Each sub-object has a boolean fetch property which can be checked to (suggest) eager loading of that object.

Fields can be dragged off the inspector itself and placed on the canvas to produce an appropriate editor.

Each data-object or sub-object has a virtual form property which can be dragged off and placed on the canvas to produce a Form for this object (a group box auto-populated with editors).

Each data-object or sub-object has a virtual grid property which can be dragged off and placed on the canvas to produce a Grid for this object.

There is an Operation component which can be associated with one or more Data components to perform batch operations.

There are NavigationBar (first, previous, next, last) and OperationBar (New, Save, Cancel toolbar) widgets that can be placed anywhere and associated with one or more Data components.
